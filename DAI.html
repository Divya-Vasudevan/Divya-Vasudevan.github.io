DAI Alarms Console

ux design + dashboards + prototyping

Designing the alarms feature to analyze and monitor Dynamic Ad Insertion

My Role

UX design and prototyping


Duration

3 weeks

The Goal

Add a DAI monitoring dashboard to an existing analytics product.

What is DAI?

Dynamic ad insertion(DAI) is the act of adding advertisements to a program asset in real-time, rather than pre-programming it.


What does this project involve?

  • Create a dashboard to facilitate the monitoring of DAI - keep track of usage metrics and ensuring that the functionality is working accurately.

  • Ensure that the UI is similar to the parent analytics product

Requirement Gathering and Initial Sketches

  • Went through the engineering teamâ€™s requirements document that listed the necessary features.

  • Studied the existing UI to ensure that the new dashboard would be familiar to users.

  • Spoke to stakeholders to understand how they would use the dashboard.

  • Sketched out low fidelity wireframes for the dashboard and individual features.

Design: Version 1

Initial designs of major features were used as a visual aid to further discussions about functionality and understand requirements better

Since I already had a guide for the interface design, I moved from low fidelity wireframes to a high fidelity first draft. I only included some major features that had clear directions on how and where it would be used. This draft also helped act as a visual aid to discuss clarifications about the remaining functionality with the stakeholders. The discussion was centered around:

  • A detailed look into how the dashboard would be used.

  • Identifying areas of improvement which could mean additional features or edits to existing ones.

  • Ensuring that the development team and stakeholders were on the same page with respect to the functionality and use of the features designed.

User Flow

With the discussion clarifying the needs of the stakeholders I was able to get a clear understanding of all the features and develop a user flow.

Final Design

Based on the information architecture and the previous feedback, I iterated the designs to complete the final version. I also created a clickable prototype for stakeholder presentation. The prototype would also help developers to understand the user flow more easily.
